# 灰度传感器组数据模型

## 职责边界

`platform_lsensor` 封装平行安装的二值灰度传感器组，负责采样、按物理
安装位置换算线中心，并以 mm 为单位向上提供一致快照。丢线恢复、路口
判断和转向控制属于 `Server_Tracking`，不放入 BSP 接口。

平台最多保留 8 个稳定物理槽位，实际装配路数由板级映射决定，未装配槽位
不启用。本产品当前装配 8 路（PA0–PA7，槽位 i = PAi，见核心引脚分配表）。
GPIO 映射、有效电平和探头间距最终以资源分配表及板级映射
（`board_lsensor_map.h`）为准。

## 坐标与几何约定

- 从车尾向车头看，槽位编号从左到右递增。
- 灰度阵列的几何中心为 `0 mm`，左侧为负，右侧为正。
- 探头位置/间距是 BSP 板级配置（`board_lsensor_map.h` 位置表），不随
  快照上报。
- 八路等间距阵列的探头位置依次为 `-3.5d、-2.5d、-1.5d、-0.5d、0.5d、
  1.5d、2.5d、3.5d`，其中 `d` 为相邻探头中心距。

## 向上数据

| 字段 | 含义 |
|---|---|
| `line_offset_mm` | 检测线相对阵列中心的偏差，右正左负，单位 mm，仅 TRACKING 有效 |
| `track_state` | 无线、可跟踪或结果歧义 |
| `run_state` | 传感器组是否处于可采样状态 |

> 2026-07-13 契约评审（D4）：`sensors[]`、`line_map`、`sensor_count` 不再
> 随快照上报，仅作为 BSP 内部与调试数据（RTT 日志可见）。

`line_offset_mm` 按所有命中探头物理位置的算术平均值计算：

```text
line_offset_mm = sum(hit_sensor_position_mm) / hit_count
```

连续的一簇命中点为可跟踪结果。没有探头命中时，读取本身仍可成功，但
`track_state` 为无线，`line_offset_mm` 无效。出现多个不连续命中簇或全路
命中时标记为歧义。位图不随快照上报；路口等业务语义将来由 Server 经新增
查询接口获取（见《灰度传感器模块设计》9.4）。

## 错误语义

- `platform_err_t` 只表达接口调用和硬件采样是否成功。
- 无线和歧义属于成功采样后的检测结果，不返回平台错误。
- 读取失败时不得修改调用方提供的输出快照。
