# 灰度传感器组数据模型

## 职责边界

`platform_lsensor` 封装平行安装的二值灰度传感器组，负责采样、按物理
安装位置换算线中心，并以 mm 为单位向上提供一致快照。丢线恢复、路口
判断和转向控制属于 `Server_Tracking`，不放入 BSP 接口。

平台最多保留 8 个稳定物理槽位。实际装配路数由板级映射决定，例如七路
阵列使用槽位 0～6，剩余槽位不启用。GPIO/ADC 映射、有效电平和探头间距
最终以资源分配表及板级映射为准。

## 坐标与几何约定

- 从车尾向车头看，槽位编号从左到右递增。
- 灰度阵列的几何中心为 `0 mm`，左侧为负，右侧为正。
- `next_spacing_mm` 表示当前探头中心到下一有效探头中心的距离，单位 mm；
  最后一个有效探头填 0。
- 七路等间距阵列的探头位置依次为 `-3d、-2d、-d、0、d、2d、3d`，
  其中 `d` 为相邻探头中心距。

## 向上数据

| 字段 | 含义 |
|---|---|
| `sensors[]` | 各稳定槽位的本次检测状态和安装间距 |
| `line_map` | 位 n 对应槽位 n，置 1 表示该路检测到线 |
| `sensor_count` | 实际启用的连续槽位数 |
| `line_offset_mm` | 检测线相对阵列中心的偏差，右正左负，单位 mm |
| `track_state` | 无线、可跟踪或结果歧义 |
| `run_state` | 传感器组是否处于可采样状态 |

`line_offset_mm` 按所有命中探头物理位置的算术平均值计算：

```text
line_offset_mm = sum(hit_sensor_position_mm) / hit_count
```

连续的一簇命中点为可跟踪结果。没有探头命中时，读取本身仍可成功，但
`track_state` 为无线，`line_offset_mm` 无效。出现多个不连续命中簇或全路
命中时标记为歧义，并保留 `line_map` 供 Server 判断路口等业务语义。

## 错误语义

- `platform_err_t` 只表达接口调用和硬件采样是否成功。
- 无线和歧义属于成功采样后的检测结果，不返回平台错误。
- 读取失败时不得修改调用方提供的输出快照。
