# 差速底盘里程计模块设计

## 1. 模块边界

`Sever_diff_dri_odom` 位于 `05_Server`，负责把左右轮原始累计编码器
计数解算为平面位姿和底盘速度。模块只依赖
`platform_pmotor_get_state()`，不得访问 HAL、编码器 port 或电机驱动内部状态。

本模块只做里程计，不负责电机目标下发、闭环调速、循迹决策或 IMU 融合。

## 2. 数据与配置

- 输入：左右轮 `platform_pmotor_state_t.position` 原始累计 tick。
- 装配：左右电机物理槽位、每 tick 位移、轮距及前进方向符号。
- 输出：`x/y/yaw` 平面位姿和 `vx/vy/w` 底盘速度快照。
- 单位：长度使用 mm，时间使用 s，角度使用 degree。
- 差速约束：`vy_mm_s` 恒为 `0`。

每 tick 位移由轮径、减速比和编码器分辨率标定后，通过配置层传入；
模块内不得硬编码板级参数。里程计必须对原始累计 tick 做差分，不得使用
电机速度环的滤波转速积分。
解算任务周期由 `APP_ODOM_PERIOD_MS` 统一配置，任务使用 OSAL
定周期延时维持绝对唤醒时刻，不得使用普通相对延时作为正常节拍。
解算任务资源由服务头文件中的 `SERVER_ODOM_STACK_SIZE` 和
`SERVER_ODOM_TASK_PRIO` 固定配置，当前分别为 `2048U` 字节和 `24U`；
不得通过 `server_odom_cfg_t` 传入，以免装配接口暴露 OSAL 任务类型。

### 2.1 坐标系与解算公式

坐标系与 IMU 接口保持一致：航向角为 `0 deg` 时车头沿世界坐标
`+X` 方向，世界坐标 `+Y` 与车体右侧一致，航向角顺时针为正。

每次解算先对原始累计 tick 做整数差分，再应用轮向符号和标定比例：

```text
delta_l = (left_now - left_last) * left_sign
delta_r = (right_now - right_last) * right_sign
dist_l  = delta_l * left_mm_tick
dist_r  = delta_r * right_mm_tick
dist    = (dist_l + dist_r) / 2
dyaw    = (dist_l - dist_r) / track_mm
```

其中 `dyaw`、`yaw_mid` 和参与三角函数的 `yaw` 均使用 rad；对外发布的
`yaw` 和 `w` 再换算为 degree。位姿采用中点航向积分，角速度直接由未
归一化的本周期角增量计算：

```text
yaw_mid = yaw + dyaw / 2
x       = x + dist * cos(yaw_mid)
y       = y + dist * sin(yaw_mid)
yaw     = normalize(yaw + dyaw)
vx      = dist / dt
vy      = 0
w       = dyaw / dt
```

首次成功读到左右轮位置时只建立差分基线，不积分累计位置。任一侧
读取失败时不得推进基线；恢复后的速度使用从上次有效样本起累计的
周期时长。内部使用浮点位置累计，发布 `int32_t` 坐标时再四舍五入，
避免每周期亚毫米位移被截断丢失。

## 3. 生命周期与状态约束

生命周期为 `init -> start -> process -> stop -> deinit`：

- `init`：校验并保存配置，姿态清零，不访问电机计数。
- `start`：读取左右轮当前累计 tick 作为基线，避免把停机前运动误计入。
- `process`：按调用方提供的实际间隔更新一次状态；间隔必须大于零。
- `stop`：停止解算并将速度清零，保留位姿和装配配置。
- `deinit`：清除内部状态，回到未初始化状态。

`set_pose` 可在已初始化状态下校准位姿，并将瞬时速度清零；运行中调用时
不得破坏当前编码器差分基线。读取或解算失败时，不发布半更新状态。

## 4. 调用与并发约束

里程计状态只允许在任务上下文访问，禁止在 ISR 中调用。
`server_odom_task` 提交状态、`set_pose` 校准位姿和 `get` 读取快照
必须使用同一把 OSAL Mutex 保护。

电机读取与左右轮增量计算在锁外完成；依赖当前姿态的坐标变换、状态
快照提交和复制在锁内完成。`set_pose` 每次校准都递增位姿代次，任务在
双轮采样前后检查该代次；发现采样跨越校准边界时必须丢弃该样本并重建
编码器基线，禁止把校准前的历史增量叠加到新位姿。

Server 只依赖 OSAL 接口，不得 include FreeRTOS 实现头文件。
`deinit` 只能在调用方停止其他 Odom API 调用后执行，避免销毁仍有
等待者的 Mutex。
