## 一、模块定位与范围

| 项 | 内容 |
|---|---|
| 器件 | 8 路平行二值灰度探头（数字输出，型号/间距待实测），PA0–PA7，GPIO 输入（ioc 标签 `PA0_LSENSOR_0`…`PA7_LSENSOR_7`） |
| 本模块提供 | 一次调用同步取"同拍"快照：线中心偏差 mm（右正左负）+ 识别状态（NO_LINE/TRACKING/AMBIGUOUS）+ 运行状态 |
| 本模块**不**提供 | 丢线恢复、路口判断、循迹闭环 —— 归 `Server_Tracking`；`line_map` 不进上行契约（D4），仅作内部/调试数据 |

| 执行上下文 | `platform_lsensor_read()` 仅任务/主循环上下文调用，单消费者约定；无 ISR、无后台采样、无周期检测（D1/D5） |

## 二、已拍板决策（2026-07-13 评审）

| # | 决策 | 内容与理由 |
|---|---|---|
| D1 | **不上 DMA，同步阻塞读** | 上层调用 read → 经 MCU 接口读一拍 IDR → 就地算完上报。8 bit 数据量、百 Hz 级控制环，直读足够；DMA 是后续优化方向（演进档案见 9.1） |
| D2 | **合同绑数据事实，不绑搬运手段** | MCU 层合同 = "给我这一拍的 8 路电平位图"。DMA / 定时采样 / 直读都是 `gpio_port` 私有实现，升级引擎不动 BSP 与合同 |
| D3 | **8 路 PA0–PA7** | ioc 已配 8 路 GPIO_Input（PA0 键名为 `PA0-WKUP`）；**bit i = PAi = 槽位 i**，IDR 低字节整字节对应 |
| D4 | **上行契约维持三字段/三接口** | 尊重 410676f 的极简化；《传感器数据模型》同步删去 sensors[]/line_map/sensor_count 上报；Server 将来需要路口语义时走新增查询接口（9.4） |
| D5 | **单实例、无 handler，且无周期采样** | 全车仅一个循迹模块、同步读取（D1）：电机式 handler 的两大职责——多实例容器、控制节拍调度——均不存在 → driver 核心收敛为 `bsp_lsensor_driver.c/.h`，契约适配独立放在 `platform_lsensor.c` |
| D6 | **数据源依赖注入** | driver 单实例持配置（`lsensor_drv_cfg_t`），硬件读取经 cfg 内 `get_map` 函数指针注入（返回原始电平位图）；契约实现区用适配函数包 `platform_iogroup_read` 后注入。收益：`.h` 不见任何平台接口，PC 单测可注入假数据源直测 read 归一路径（对齐 pmotor 注入范式） |

## 四、接口定义

### 4.1 platform_io_group.h（合同，WP1 落地）

```c
#include "platform_error.h"
#include "platform_type.h"

#define PLATFORM_IOGROUP_ID_UNUSED 0xFFFFFFFFU

typedef uint32_t platform_iogroup_id_t;

/**
 * @brief  读取逻辑输入组当前一拍电平位图
 * @param  id     逻辑输入组标识
 * @param  bitmap 位 i = 组内第 i 路原始电平（1=高），高位补 0，不可为 NULL
 * @retval PLATFORM_ERR_OK / PLATFORM_ERR_PARAM / PLATFORM_ERR_FAIL
 * @note   组到端口/引脚/位序的映射只存在于 gpio_port（唯一依据：资源分配表）
 */
platform_err_t platform_iogroup_read(platform_iogroup_id_t id, uint32_t *bitmap);
```

语义细则：

- 返回**原始电平**；压线有效电平由 driver 依据 cfg.active_low 归一（归属分界见 5.1）。
- 单次读取天然"同拍"（port 内为一条 IDR 读指令 + 位序打包）。
- 本函数任意上下文可调用（纯寄存器读）；上层契约的上下文限制另行约定。

### 4.2 演进预留（当前评审明确**无周期采样需求**，本节仅存档）

若将来需要采样引擎，**只增函数、不改 4.1**，BSP 现有代码零改动：

```c
/* 定频采样引擎：port 内部自选 TIM+DMA 或 platform_timer ISR 轮询 */
platform_err_t platform_iogroup_sample_start(platform_iogroup_id_t id, uint32_t sample_hz);
platform_err_t platform_iogroup_sample_stop (platform_iogroup_id_t id);
/* 取最新一拍 / 最近 n 拍（新→旧），滤波归上层 */
platform_err_t platform_iogroup_get_latest(platform_iogroup_id_t id, uint32_t *bitmap);
platform_err_t platform_iogroup_get_window(platform_iogroup_id_t id,
                                           uint32_t *bitmaps, uint32_t n, uint32_t *filled);
```

- 环形覆盖是设计行为，**无 overrun 错误路径**（只关心最近窗口）。
- 软引擎受 `platform_timer` 1 ms 粒度限制（≤1 kHz）；DMA 引擎无此限但受 9.1 资源约束。
- 若将来需要事件/时间戳，按 tim_port 的 callback+ctx 范式增加 `register()`，同样是增量扩展。

### 4.3 driver 合同（模块内部，对上只有 platform_lsensor.h；已落地）

```c
/* bsp_lsensor_driver.h —— 只 include platform_type.h，不见平台接口 */
typedef enum {
    LSENSOR_DRV_OK        = 0,
    LSENSOR_DRV_ERR_PARAM = 1,
    LSENSOR_DRV_ERR_STATE = 2,    /* 未初始化 */
    LSENSOR_DRV_ERR_IO    = 3,    /* 注入数据源读取失败 */
    LSENSOR_DRV_RESERVED  = 0xFF
} lsensor_drv_status_t;

typedef enum {
    LSENSOR_DRV_NO_LINE = 0,
    LSENSOR_DRV_TRACKING,
    LSENSOR_DRV_AMBIGUOUS
} lsensor_drv_track_t;

/* 注入的数据源：读一拍8路原始电平位图，位i=槽位i，1=高电平 */
typedef lsensor_drv_status_t (*pf_lsensor_get_t)(uint8_t *raw_map);

typedef struct {
    float   positions_mm[LSENSOR_DRV_SLOT_MAX]; /* 槽位 i 坐标，右正左负 */
    uint8_t enabled_mask;     /* 位 i=1 装配；0 为断点 */
    uint8_t active_low;       /* 1=压线为低电平 */
    pf_lsensor_get_t get_map; /* 外部依赖注入，不可为 NULL */
} lsensor_drv_cfg_t;

typedef struct {
    float               offset_mm;  /* 仅 TRACKING 有效，其余置 0.0f */
    lsensor_drv_track_t track;
} lsensor_drv_result_t;

lsensor_drv_status_t lsensor_drv_init(const lsensor_drv_cfg_t *cfg);
lsensor_drv_status_t lsensor_drv_deinit(void); /* 最小实现：清状态 */
/* read = get_map 取数 → 归一（active_low 取反、& mask）→ 内部簇分析 */
lsensor_drv_status_t lsensor_drv_read(lsensor_drv_result_t *out);
/* 簇分析内核为 .c 内 static；对外只经 read 暴露，单测走注入假源 */
```

算法规格（WP3 单测依据）：

1. 归一在 `lsensor_drv_read` 内完成：`active_low ? ~raw : raw`，再 `& enabled_mask`；
   内核逐槽位按 enabled_mask 过滤，天然防御未裁剪输入。
2. 槽位 0→7 扫描**连续命中段（run）**；disabled 槽位视作断点。
3. 判定：run 数 = 0 → NO_LINE；run 数 ≥ 2 → AMBIGUOUS；全部使能槽位命中 → AMBIGUOUS（全路触发）；
   否则 TRACKING，`offset = mean(命中槽位 positions_mm)`（《传感器数据模型》公式）。
4. 输出确定性：非 TRACKING 时 `offset_mm` 置 0.0f，调用方不得使用（契约已注明）。
5. driver 核心**无跨拍数据状态**（仅持只读 cfg 副本），read 可重入；
   PC 单测经 `lsensor_drv_read` + 注入假 get_map 全链路表驱动（含极性与 IO 失败路径）。
6. 几何与注入合法性（cfg/get_map 非 NULL、enabled_mask 非 0、启用槽位坐标严格递增）
   收口在 `lsensor_drv_init()`；重复 init 直接覆盖，ALREADY 语义由契约实现区负责。

### 4.4 契约实现行为规格（platform_lsensor.c）

| 接口 | 行为 |
|---|---|
| `init` | 组装 cfg（板级表几何/极性 + `get_map` = 包 `platform_iogroup_read` 的静态适配函数）→ `lsensor_drv_init` → 成功置 ACTIVE；重复 init 返回 `ALREADY_INIT`。**无 start/stop**（契约冻结）：无引擎、无启停语义，自洽 |
| `read` | 未初始化 → `NOT_INITIALIZED`；NULL → `PARAM`；调 `lsensor_drv_read`：`ERR_IO` → `FAIL` 且**不改输出**（driver 已保证失败不写 out）；成功则 track 映射为 `PLATFORM_*` 并装快照，`run_state` 恒为当前模块状态 |
| `deinit` | `lsensor_drv_deinit` → 置 OFF；再次使用必须重新 init |
| ERROR | 仅 init 阶段配置/硬件校验失败进入（对齐 .h 注释"理论上不会进入"） |

## 五、关键设计决策

### 5.1 位序与有效电平的归属分界

- **位序（PAx → bit i）= MCU 事实**，收 `gpio_port` 组表，唯一依据是资源分配表；
- **有效电平（压线 = 0/1）= 器件事实**，收 BSP 板级表 → `cfg.active_low`，
  归一动作在 `lsensor_drv_read` 内执行。

对齐电机模块 `output_invert`/`encoder_invert` 的归属哲学：芯片的事归 port，器件的事归 BSP。

### 5.2 读时计算、仅持只读配置

driver 核心无跨拍数据状态（只有 init 拷贝的 cfg 副本）→ 无并发共享、天然可单测。
若实车毛刺明显，后续在 driver 内加窗口消抖（引入跨拍状态时同步补并发约定），上下契约都不动。

### 5.3 line_map 的观测通道

不进上行契约（D4）。待 `04_Debug_Tool/RTT/` 落地后，由契约实现区以 LOGD 输出
（模块 tag `LSEN`，宏 `LSENSOR_LOG_EN` 默认 0）供实车对拍；日志关闭时热路径零开销，
遵守《日志规范》。

### 5.4 上下文与并发纪律

read 仅任务/主循环上下文、单消费者；多任务消费时由 Server 自行串行化。
无 ISR 写方 → 无跨上下文共享数据，电机文档 5.7 的原子性问题在本模块不存在。

## 六、数据流（一次 read）

```
Server 任务 ── platform_lsensor_read(&st)
  └ 契约实现区：状态/入参校验
      └ lsensor_drv_read(&res)                      ── driver 核心
          ├ cfg.get_map(&raw)  ── 适配函数 ── platform_iogroup_read ── gpio_port：一条 IDR 读
          ├ line_map = 归一(raw, active_low) & enabled_mask
          └ 内部簇分析(line_map) → res               ── run 扫描 + 均值偏差
      └ st = { res.offset_mm, res.track → PLATFORM_*, ACTIVE }   成功才写出
```

耗时量级：一次寄存器读 + 8 次循环 ≈ 微秒级；对 100~200 Hz 的 Server 节拍余量巨大。

## 九、待定项与演进档案

| # | 事项 | 说明 |
|---|---|---|
| 9.1 | **DMA 引擎演进档案**（D1 的后续路径） | F411 上 DMA1 到不了 GPIO（总线矩阵），搬 IDR 必须 **DMA2**；能触发 DMA2 的定时器只有 **TIM1**（F411 无 TIM8），而 TIM1 已被电机 PWM 占用（CH1/CH2 = PA8/PA9，ARR=10000）。故将来上 DMA 需先做资源重排决策（电机 PWM 挪 TIM3/TIM4），或接受与 PWM 共享 TIM1_UP（采样率绑死 ≈5 kHz，且 `HAL_TIM_PWM_Stop` 清 CEN 会连带停采样，需跨 port 契约）。中间档：软采样引擎（gpio_port 挂 platform_timer 1 kHz）即可得等间隔采样 + 窗口滤波，无资源冲突。无论哪档，合同不变（4.2） |
| 9.2 | 探头间距 d / 有效电平 | 实测后填板级表与资源分配表 |
| 9.3 | 消抖窗口是否需要 | WP4 实车看毛刺，需要则按 5.2 演进 |
| 9.4 | Server 路口语义的信息来源 | 契约不含 line_map（D4）；将来经新增查询接口（如 `platform_lsensor_get_map`）增量扩展 |
| 9.5 | 历史草稿清理 | `platform_line_sensor/` 下 `platform_lsensor.txt`、`platform_lsensors.txt` 与冻结契约不一致，建议删除或移出接口目录归档 |
