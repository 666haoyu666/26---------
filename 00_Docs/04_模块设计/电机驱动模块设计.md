# 电机驱动模块设计（pwm_motor · PWM 直驱 + 编码器闭环）

> 状态：**设计评审稿 v4**（未落代码）。配套文档：《底盘服务设计》（待出，本文档只管**电机能力**）。
> 参考实现：幻尔 RosRobotControllerM4 `Hiwonder/Peripherals/encoder_motor.*` + `Portings/motor_porting.c`（仅参考机制，不照抄代码）。
> 回调范式：参考立芯工程 USART port 的“port 保存回调表、HAL 回调按实例分发”机制，以及 MPU6050 driver 初始化时主动注册内部回调的机制；只借鉴注册/转发思路，不照搬跨层全局 `extern`。
> 用途：① 接口评审定稿；② 按第八节工作包分工。
> 命名约定：模块名 **pwm_motor**（区分特征是 PWM 直驱这种驱动方式；编码器是标配不进名字）。

## 一、模块定位与范围

| 项 | 内容 |
|---|---|
| 器件 | 直流减速电机 + AB 相霍尔编码器，PWM 类驱动板（双通道 PWM 或 PWM+DIR，见 5.4） |
| 本模块提供 | 单电机速度闭环（目标 rps → 输出）、转速测量、**原始累计位置计数**（供里程计）、多电机统一管理与同步调度；handler 自行把内部控制回调注册到 `tim_port` |
| 本模块**不**提供 | 底盘运动学（差速/麦轮解算）、里程计解算、左右轮目标的成组加锁/原子更新 —— 归 `service_chassis`（另文档）；Service 不参与电机速度环调度 |
| 对上暴露 | `platform_pmotor` 模块级能力接口（**稳定物理 ID + 函数**，不暴露电机对象和 `process`），唯一实现者为 handler |
| 依赖 | `platform_mcu`（PWM/IO/编码器/控制节拍抽象）；PID 为 `pwm_motor` driver 私有实现；**不直接依赖 HAL**（B 方案，已定） |
| 执行上下文 | 速度环运行在**高优先级定时器中断**上下文（见 5.3）；CubeMX IRQ 只做 HAL 胶水，HAL 回调与注册分发收口 `impl_mcu/tim_port` |

## 二、参考实现机制分析（幻尔编码器电机）

### 2.1 结构：对象 + 函数指针注入（与本项目范式天然契合）

```
encoder_motor.c/.h        硬件无关对象：counter/overflow_num/tps/rps/current_pulse
                          + pid_controller + ticks_per_circle/rps_limit(参数)
                          + set_pulse 函数指针（porting 注入）
motor_porting.c           适配：4 个 motorX_set_pulse(TIM PWM)、编码器 TIM 启动、参数表装载
motors_param.h            参数表：JGB520/JGB37/JGA27/JGB528 的 tpc/PID/rps_limit
stm32f4xx_it.c            TIM2/3/4/5 溢出中断 ±overflow_num；TIM7@10ms 跑测速+PID
```

### 2.2 关键机制（本设计保留）

| 机制 | 细节 | 本设计归宿 |
|---|---|---|
| **不清零增量式计数** | TIM 计数器自由跑（0~60000 回绕），**读取时从不清零**；溢出中断按方向 ±`overflow_num`，读数拼成 int64 绝对计数。优点：读/清之间不丢脉冲（清零式每周期丢一点、积成系统性漂移），且计数本身就是位移积分，可直接喂里程计 | 收进 `impl_mcu` 编码器适配（见 5.2） |
| 测速 | 两次读数做差：`tps = Δcount/period` 一阶低通（新 0.9 / 旧 0.1），`rps = tps / ticks_per_circle`。**低通只作用于速度估计，不作用于计数值** | driver 纯逻辑 |
| 速度环 | 每电机独立控制器状态，输出限幅 ±1000；具体 PID 形式经实验定稿 | `pwm_motor` driver 私有逻辑，不依赖公共 PID 模块 |
| 死区处理 | \|pulse\| < 250 时置 0（PWM 过小电机只嗡嗡不转） | driver 纯逻辑，死区值进配置 |
| 高优先级定时中断跑控制环 | TIM7@10ms 定时中断执行测速+PID，实时性好、不依赖 RTOS | **保留执行上下文**；handler 自注册内部控制回调，经 `tim_port` 分层路由（见 5.3，对比 P1） |
| 参数表 | 每型号一组 tpc/PID/rps_limit 宏 | `00_Config`（见第七节，tpc 直接定值） |

### 2.3 参考实现问题清单（我们要避开的坑）

| # | 问题 | 后果 | 本设计对策 |
|---|---|---|---|
| P1 | 控制链**写死在 vendor 中断文件**：`stm32f4xx_it.c` 里 extern 对象数组、直调驱动函数（在中断里跑控制环本身**不是错**，实时性反而是优点） | 生成码与业务纠缠，换芯片/重生成即冲突；对象全局裸奔 | CubeMX IRQ 只保留 `HAL_TIM_IRQHandler()` 胶水；`tim_port` 保存 callback + ctx 并分发，motor handler 初始化时主动注册自己的控制回调（见 5.3） |
| P2 | 4 份 `motorX_set_pulse` 复制粘贴 | 已产出真实 bug：motor4 编码器启动误写 `htim4`（motor_porting.c:111，实为 htim3） | **表驱动**：`impl_board` 一张映射表，driver/handler 代码各只有一份（见 5.4） |
| P3 | `overflow_num`（编码器 ISR 写）与 CNT 读取非原子 | 溢出瞬间读数可能跳 ±60000 | 原子读策略收口在 `impl_mcu`（见 5.2/5.6） |
| P4 | PID"增量式"实现有瑕疵：算了 `proportion = err - prev_err` 却没用，实际输出 `kp*err` 再被累加 | P 项实际表现成积分行为，参数语义混乱（JGA27 甚至配负增益） | PID 私有收进 `pwm_motor` driver；具体形式经实验定稿，但实现必须明确位置式/增量式、采样周期、饱和与复位语义，禁止混合形式 |
| P5 | 魔数散落：60000、±1000、死区 250、LPF 0.9 硬编码在逻辑里 | 换电机/换定时器要翻代码 | 全部参数化，收口 `00_Config`（见第七节） |
| P6 | motor3/4 换向前先 `PWM_Stop` 再 `Start`，motor1/2 又没做 | **疑似 H 桥换向保护**（防换向瞬间双通道同时有效直通短路），但四个电机行为不一致——若是保护则不完整；且靠 Stop/Start 副作用实现，意图不可读 | 换向保护做成统一的**写序合同**：PID 输出先限幅校验，换向时在同一次输出函数内严格执行“旧输出归零 → 改方向/切通道 → 写新占空比”；零值必须在物理输出端先生效（见 5.4） |

### 3.1 driver / handler 职责切分

| 层 | 文件 | 职责 | 不做 |
|---|---|---|---|
| **driver** | `bsp_pwm_motor_driver.c/.h` | **单电机**纯逻辑：输入（当前原始 count, period）→ 内部差分 → 测速 LPF → 私有 PID → 死区/限幅 → 输出带符号 duty；维护每电机独立控制器状态、rps 与原始 position | ❌ IO、❌ 知道自己是几号电机、❌ include platform_mcu/HAL、❌ 依赖公共 PID 模块 |
| **handler** | `bsp_pwm_motor_handler.c/.h` | **多电机容器**：按映射表实例化 driver；初始化时把统一控制回调注册到 `tim_port`；每拍遍历全部电机完成同步采样/计算/输出；带符号 duty → 双通道 PWM 或 PWM+DIR 写序；实现 `platform_pmotor.h` 全部接口，是本模块对上的**唯一入口** | ❌ 重复实现控制算法、❌ include HAL、❌ 让 Service 编排速度环 |

> 与 ZDT 串口电机 handler 的对照：那里 handler 还要做共享总线仲裁与 TX/RX 线程；PWM 电机每路独占外设，**无总线仲裁、无线程**，所以本 handler 的份量集中在"容器 + 同步调度 + 输出策略"。若未来换成共享总线电机，仲裁逻辑就落在这层，`platform_pmotor.h` 不动。

## 四、接口草案（WP1 评审对象）

> 以下为**形状草案**，字段/命名以 WP1 评审定稿为准。全部返回 `platform_err_t`（成功=0），入参校验在 handler 入口做（if + return，不封装）。

### 4.1 platform_pmotor.h（稳定物理 ID + 模块级函数，不暴露电机对象）

```c
/* 稳定物理电机槽位：Platform ABI 不随差速/四驱产品形态改变。
   LEFT/RIGHT/LF/RF/LR/RR 等轮位语义由 service_chassis 映射。 */
typedef enum {
    PLATFORM_PMOTOR_0 = 0,
    PLATFORM_PMOTOR_1,
    PLATFORM_PMOTOR_2,
    PLATFORM_PMOTOR_3,
    PLATFORM_PMOTOR_ID_MAX         /* 固定容量；未装配槽位在 board map 标 disabled */
} platform_pmotor_id_t;

typedef enum {
    PLATFORM_PMOTOR_UNINIT = 0,
    PLATFORM_PMOTOR_READY,
    PLATFORM_PMOTOR_RUNNING,
    PLATFORM_PMOTOR_STOPPED,
    PLATFORM_PMOTOR_FAULT
} platform_pmotor_run_state_t;

typedef struct {
    float                      target_rps;   /* 当前目标转速 */
    float                      rps;          /* 滤波后输出轴转速 */
    int64_t                    position;     /* 原始累计 tick：仅符号归一化，不滤波、不清零 */
    int32_t                    applied_duty; /* 已应用带符号占空比，-1000~1000 */
    platform_pmotor_run_state_t run_state;   /* 模块运行状态 */
    uint32_t                   fault_flags;  /* 锁存故障位，具体位定义在 WP1 定稿 */
} platform_pmotor_state_t;

/* 生命周期五段对齐（模块级，一次管全部电机） */
platform_err_t platform_pmotor_init(void);     /* 装配、注册 tick callback，不启动定时器 */
platform_err_t platform_pmotor_start(void);    /* 从零目标/零控制器状态启动控制节拍 */
platform_err_t platform_pmotor_stop(void);     /* 失能全部驱动，电机滑行，保留资源/position */
platform_err_t platform_pmotor_deinit(void);

/* 能力接口 */
platform_err_t platform_pmotor_set_rps(platform_pmotor_id_t id, float rps); /* 内部按 rps_limit 限幅 */
platform_err_t platform_pmotor_get_state(platform_pmotor_id_t id, platform_pmotor_state_t *out);
```

> 对象四要素（struct/ops/ctx/state）退到 driver/handler 内部实现；平台接口只暴露能力、稳定 ID 与状态。控制 `process` 是 handler 注册给 `tim_port` 的私有回调，不是上层 API。`set_rps` 只保证单电机目标更新；左右轮成组加锁由 `service_chassis` 负责。

> handler 入口统一校验 ID 范围、槽位 enabled、输出指针与浮点合法性。`set_rps` 对 NaN/Inf 返回 `PLATFORM_ERR_PARAM`；有限但超过 `rps_limit` 的目标按正负方向限幅。

### 4.2 platform_mcu 最小接口

```c
/* platform_pwm.h —— ch 为逻辑通道号，由映射表落到 TIM/CH */
platform_err_t platform_pwm_start(uint32_t ch);
platform_err_t platform_pwm_stop(uint32_t ch);
platform_err_t platform_pwm_set_duty(uint32_t ch, uint32_t duty); /* 千分比 0~1000 */
/* 换向路径要求 duty=0 在改方向前已对物理输出生效；若使用 preload，Impl 必须通过
   立即失能、强制提交或等价机制满足该合同，不能让 0→新值合并成一次最终写。 */

/* platform_io.h —— 方向引脚用 */
platform_err_t platform_io_write(uint32_t io, uint8_t level);

/* platform_encoder.h —— id 为逻辑编码器号 */
platform_err_t platform_encoder_start(uint32_t id);
platform_err_t platform_encoder_stop(uint32_t id);
platform_err_t platform_encoder_get_count(uint32_t id, int64_t *count);
/* 语义：返回“已做 64 位溢出扩展、已保证原子性”的【原始累计计数】，不清零、不滤波；
   定时器位宽、溢出、方向位全部收在 impl_mcu，上层不感知。 */

/* platform_timer.h —— 逻辑定时器 callback + ctx 注册机制；具体命名 WP1 定稿 */
typedef void (*platform_timer_cb_t)(void *ctx);
platform_err_t platform_timer_register(uint32_t id, platform_timer_cb_t cb, void *ctx);
platform_err_t platform_timer_unregister(uint32_t id);
platform_err_t platform_timer_start(uint32_t id, uint32_t period_ms);
platform_err_t platform_timer_stop(uint32_t id);
/* 语义：motor handler 在 init 时注册私有控制回调；tim_port 只保存并分发 opaque callback
   + ctx，不 include motor 头文件。cb 在高优先级定时器 ISR 上下文执行，必须有界、
   无阻塞、无日志。CubeMX IRQ 只进 HAL；HAL callback 与实例分发在 tim_port。 */
```

### 4.3 单位与语义约定

| 量 | 单位/范围 | 约定 |
|---|---|---|
| 转速 | rps（输出轴，float，带符号） | 对上唯一速度语义；**滤波后**（控制/显示用）；rpm/线速度换算归底盘服务 |
| 位置 | tick（int64，带符号累计） | **原始值：仅按 encoder_invert 归一化符号，不滤波、不清零**；里程计由底盘服务对 position 差分；tick→mm 换算归底盘服务（需轮径） |
| 占空比 | 千分比 0~1000（uint32）+ 符号由策略层消化 | 仅 handler↔platform_pwm 语义，不上浮到 platform_pmotor |
| 方向 | target/rps/position 符号 | “正=该物理电机约定正向”；`output_invert` 与 `encoder_invert` 分别校正输出和反馈，轮位到车前进的符号关系归 `service_chassis` |

### 4.4 生命周期与停机语义

| 阶段 | 状态与动作 |
|---|---|
| `init` | 校验映射/参数，实例化全部 driver；输出保持失能；注册 handler 控制回调，但控制定时器不运行；成功进入 `READY`，失败按相反顺序回滚 |
| `start` | 目标清零、PID/滤波状态复位、读取编码器基线；确认输出为 0 后使能所需 PWM 资源，最后启动控制定时器，进入 `RUNNING` |
| 内部 tick | 仅 `RUNNING` 时允许采样、计算并提交输出；其他状态直接返回，不得恢复非零输出 |
| `stop` | 先原子禁止控制输出提交并停止控制节拍，再将全部通道置 0/失能；H 桥进入高阻滑行，清目标与控制器状态，保留资源和累计 position，进入 `STOPPED` |
| `deinit` | 若仍运行先执行 stop；注销 timer callback + ctx，再释放/反初始化外设并回到 `UNINIT` |
| 内部故障 | 锁存首错与 fault flags，禁止后续输出提交，best-effort 失能全部电机进入滑行，进入 `FAULT`；向上事件回调的种类与签名留 WP1 后续定稿 |

合法主路径：`UNINIT → READY → RUNNING → STOPPED → RUNNING`，`READY/STOPPED/FAULT → UNINIT`。`stop` 设计为安全幂等；重复 `init` 返回 `PLATFORM_ERR_ALREADY_INIT`，未初始化时调用 start/set/get 返回 `PLATFORM_ERR_NOT_INITIALIZED`。stop/deinit 即使某一路失败也继续处理其余电机，最后返回首个错误。

## 五、关键设计决策

### 5.1 B 方案：驱动不见 HAL（已定）

driver/handler 只依赖 `platform_pwm/platform_io/platform_encoder/platform_timer` 接口。换主控只重做 `impl_mcu` + 映射表，电机驱动与速度环零改动。

### 5.2 不清零计数 + 溢出扩展 + 原子读，收口在 impl_mcu

沿用幻尔的**不清零增量式**：计数器自由跑，读取从不清零（清零式在读/清之间丢脉冲，对里程计是累积性漂移）。64 位扩展是**硬件计数器位宽**问题——溢出中断（±overflow）与 `get_count` 的原子性（临界区，或"读溢出→读 CNT→再读溢出，不一致重读"）都实现在 `impl_mcu/impl_encoder.c`，对上只暴露干净的 int64 原始计数。对策 P3。

### 5.3 速度环由 handler 自注册到 tim_port，在高优先级定时中断执行（已定）

小车以电机控制为核心，控制环要最好的实时性，且不应依赖"是否上 RTOS"：

- **执行上下文**：默认高优先级定时器中断（周期 `MOTOR_CTRL_PERIOD_MS`，暂定 10ms），**不创建控制任务**。主控选型后必须实测最大电机数下的 WCET、抖动与 FPU/栈开销；满足周期裕量后该决策才完成硬件验收。
- **注册所有权**：`platform_pmotor_init()` 装配完全部电机后，由 handler 把唯一的 `pwm_motor_ctrl_tick(ctx)` 注册给逻辑控制定时器；Service 不注册、不调用、也看不到该回调。
- **分层路由**：CubeMX 生成的 `TIMx_IRQHandler()` 物理保留在 `Core`，只调用 `HAL_TIM_IRQHandler()`；HAL period callback 与 handle→逻辑 timer→callback + ctx 的分发收在 `impl_mcu/tim_port.c`。tim_port 不 include motor 头，只调用已注册的 opaque callback。
- **同拍调度**：一个控制定时器只注册一个 handler 回调；handler 在一次回调里遍历全部已启用电机，而不是每个电机各注册一个 TIM 回调。
- **ISR 纪律**：回调有界、无阻塞、无日志；只做编码器采样、测速/PID、限幅和电机输出。转向环、里程计与视觉等仍归 Service，不挂入 motor handler 回调。
- **逃生通道**：若选定主控实测无法满足 ISR 裕量，可把 tim_port 注册回调改为通知专用高优先级任务；`platform_pmotor` 对上接口不变。

### 5.4 输出策略表驱动：方向控制平台化 + 显式换向保护

各家 PWM 电机驱动板对外接口不同，方向控制做成**策略 + 表**：

```c
/* 表结构定义在 bsp_pwm_motor_handler.h（谁需要谁定义），impl_board 填表 */
typedef enum {
    MOTOR_OUT_DUAL_PWM,   /* 双通道 PWM：正转 A 路占空比 B 路 0，反转反之（幻尔式） */
    MOTOR_OUT_PWM_DIR,    /* 单路 PWM + 方向引脚高低电平 */
} motor_out_type_t;

typedef struct pwm_motor_cfg pwm_motor_cfg_t; /* driver 私有参数表，只读引用 */

typedef struct {
    platform_pmotor_id_t id;
    uint8_t  enabled;       /* 该稳定物理槽位是否实际装配 */
    motor_out_type_t    out_type;
    uint32_t pwm_ch_a;    /* DUAL_PWM: 正转通道 | PWM_DIR: PWM 通道 */
    uint32_t pwm_ch_b;    /* DUAL_PWM: 反转通道 | PWM_DIR: 不用 */
    uint32_t dir_io;      /* PWM_DIR: 方向引脚   | DUAL_PWM: 不用 */
    uint8_t  output_invert;  /* 仅校正电机输出接线/方向电平 */
    uint8_t  encoder_invert; /* 仅校正编码器 AB 相计数方向 */
    uint32_t encoder_id;
    const pwm_motor_cfg_t *cfg; /* 每电机/每型号私有参数 */
} pwm_motor_map_t;
```

- handler 内两套输出策略函数按 `out_type` 分发；新驱动板 = 加一个策略枚举 + 一个策略函数 + 改表，`platform_pmotor.h` 不动。对策 P2。
- **统一入口**：PID 输出先限幅到 `[-1000, 1000]` 并完成合法性检查，再交给 handler 私有的 `pwm_motor_apply_duty(id, signed_duty)`；所有输出策略只能从该入口提交。
- **换向写序保护**（回应 P6）：若新 duty 与已应用方向异号，策略函数在**同一次 ISR 调用**中严格执行“旧有效输出写 0 → 确认零态已物理生效 → 改 DIR/切换双 PWM 有效通道 → 写入新的绝对值 duty”。同向时只更新幅值，目标为 0 时只归零，不切方向。
- **零态生效合同**：若 PWM compare 使用 preload，Impl 不能让 `0 → 新值` 在同一 update event 合并；必须以内层立即失能输出、强制提交或等价方式保证改向前输出已为 0。该保证封装在 PWM/输出策略实现中，不上浮 Service。
- **时间 guard**：V1 不强制跨控制周期等待；如果选定 H 桥数据手册明确要求 dead time，再由对应输出策略扩展非阻塞 guard 状态，不能在 ISR 忙等。
- 映射表固定按 `PLATFORM_PMOTOR_ID_MAX` 使用 designated initializer，条目数做**静态断言**；未装配槽位 `enabled=0`，访问返回 `PLATFORM_ERR_NOT_SUPPORTED`。漏掉槽位或重复资源在 init 校验时报错。

### 5.5 控制律沿用幻尔行为，但实现规范化

保留：一阶低通测速（系数可配）、±限幅，以及第一版简单死区策略：`|output| < deadzone → 0`，达到死区后直接输出。

PID 不再依赖 `99_Utils/pid`，而是封装在 `pwm_motor` driver 内；每个 driver 实例有独立参数、误差历史与积分状态。位置式/增量式等具体结构由 WP3 仿真与实车实验决定，但必须同时写清采样周期、输出饱和、stop/start/换向时的复位或冻结行为。

简单死区可作为 V1 基线。为避免阈值附近积分持续累积或来回抖动，WP3 同时保留“条件积分/抗饱和”和 `deadzone_start > deadzone_stop` 滞回方案的对比用例；是否启用由实验结果决定，不改变接口。死区/限幅/滤波系数全部参数化。对策 P4/P5。

### 5.6 里程计吃原始计数，滤波只给控制环（已定，留实验验证钩子）

**结论：`position` 一律暴露“仅做编码器极性归一化”的原始累计计数；低通只作用于 `rps`。** 理由：

1. **误差性质不同**。正交计数是纯数字量，原始计数的量化误差恒为 ±1 tick（3960 tpc + 65mm 轮 ≈ 0.05mm），**有界、不随时间累积**；而"滤波速度 × dt 积分"的误差是**累积**的——低通相位滞后使每次加减速都系统性少积/多积，循迹恰是不停变速的场景。
2. **航向是重灾区**。差速里程计的 heading 靠左右轮位移差，两路滤波滞后稍有不对称，航向漂移单调增长。
3. **滤波治不了打滑**。打滑是真实计数，滤波只会把错抹匀；差分测速的量化噪声（10ms 下 1 tick ≈ 0.025 rps 抖动）确实需要滤波——但那是 PID 反馈的需求，不是里程计的。
4. 幻尔实现与此一致：LPF 只在 `tps`，counter 原始。

**实验验证（后续做）**：同一段含加减速的直线，分别用原始差分与滤波速度积分算里程比终点误差；再加原地自旋/8 字对比航向漂移。预期滤波版误差随变速次数增长。若实验推翻，只需在底盘服务换数据源，本模块接口不动。

### 5.7 跨上下文共享数据的原子性约定

速度环在 ISR、`set_rps`/`get_state` 在任务/主循环上下文，共享数据边界如下：

| 数据 | 写方 → 读方 | 策略 |
|---|---|---|
| 单电机目标 rps | 任务 `set_rps` → 控制 ISR | handler 采用目标槽位 + 最小临界区/等价编译器可见性措施；Platform 只保证一次单电机更新，左右轮成组锁归 `service_chassis` |
| 完整 state 快照 | 控制 ISR → 任务 `get_state` | `position` 为 64 位且多个字段需同拍一致：短临界区复制整份 state，或使用序号双读校验 |
| 编码器溢出拼接 | 溢出 IRQ ↔ 控制 ISR 读 count | 收口 impl_mcu（5.2）；溢出 IRQ 抢占优先级严格高于控制 IRQ，配 pending 标志处理与双读校验 |

中断优先级规划（WP2 落实）：编码器溢出 > 控制节拍 > 其余业务中断，具体数值进 `01_资源分配表`。

## 六、数据流（一个控制拍）

```
Core/Src/stm32xx_it.c  CubeMX TIM IRQ（只调用 HAL_TIM_IRQHandler）
   ▼
impl_mcu/tim_port.c    HAL period callback → handle 映射逻辑 timer → callback + ctx
   ▼
bsp_pwm_motor_handler 私有 ctrl_tick（init 时自注册；Service 不进入此链）
   ├─ 对每个电机（同拍）：platform_encoder_get_count(id) ──→ impl_mcu（原子、原始值）
   ├─ bsp_pwm_motor_driver（纯逻辑）：count_now→Δcount→tps(LPF)→rps；私有 PID→带符号 duty
   └─ 输出策略分发：DUAL_PWM → platform_pwm_set_duty(a/b)
                    PWM_DIR  → platform_io_write(dir) + platform_pwm_set_duty(ch)
                    （换向严格执行“零→改向→写值”，见 5.4）
```

目标速度路径：Service/任务上下文 `set_rps(id, rps)` → rps_limit 限幅 → 单电机目标槽位（下一拍生效）；左右轮成组更新的锁由底盘 Service 持有。
状态读取路径：`get_state(id)` → target/rps/**原始 position**/applied_duty/run_state/fault_flags 的一致快照；里程计由底盘服务按自己的周期对 position 差分。

## 七、参数与配置（收口 00_Config）

| 参数 | 说明 | 初值参考（幻尔） | 定值方式 |
|---|---|---|---|
| `MOTOR_TICKS_PER_CIRCLE` | 输出轴每转计数，**直接定值宏**（换算式只作注释，如 /* 11 脉冲×4 倍频×90 减速比 */） | `3960` | 按实际电机标称+实测校验（WP4） |
| `MOTOR_RPS_LIMIT` | 转速限幅（略小于 100% 占空比转速） | 1.35~1.5 | 实测标定（WP4） |
| `MOTOR_PID_KP/KI/KD` | 速度环增益 | kp63/ki2.6/kd2.4 仅作起点 | 实车整定（WP4） |
| `MOTOR_PWM_DEADZONE` | 输出死区（千分比） | 250 | 实测最小启动占空比 |
| `MOTOR_SPEED_LPF_ALPHA` | 测速低通新值权重（只影响 rps，不影响 position） | 0.9 | 联调调整 |
| `MOTOR_CTRL_PERIOD_MS` | motor handler 私有控制节拍周期 | 10 | 按电机动态、编码器分辨率与 ISR WCET 定 |
| `FEATURE_MOTOR` | 功能开关（feature_config.h） | 置 1 | — |

`PLATFORM_PMOTOR_ID_MAX` 表示稳定物理槽位容量，不等于当前启用数量；差速/四驱不再改变 Platform 枚举，实际启用槽位由 board map 的 `enabled` 决定，轮位映射归 `service_chassis`。

电机具体型号未定 → 参数按“每型号一份只读 `pwm_motor_cfg_t`，每个 map 条目引用一份 cfg”组织；允许左右电机使用不同 tpc/PID/rps_limit/deadzone 标定值。具体配置文件落 `00_Config`，PID实现仍保持 driver 私有。

## 十、验收清单（模块级）

- [ ] App/Service 层 grep 不到 `stm32`/`HAL_`/TIM/通道号；Service 只见稳定 motor ID、目标与状态，不见 motor process/tick 注册
- [ ] CubeMX `TIMx_IRQHandler` 只调用 HAL；HAL period callback、handle→逻辑 timer 映射和 callback + ctx 表收口 `tim_port`，且 tim_port 不 include motor/Service
- [ ] `impl_bsp/pwm_motor/` 全目录 grep 不到 HAL；driver 文件另 grep 不到 `platform_pwm/platform_io/platform_encoder`（IO 只许在 handler）
- [ ] handler 在 init 注册唯一控制回调、start 后放行、stop 后禁止提交、deinit 注销；Service 不参与控制链
- [ ] `position` 为原始累计计数：不滤波、不清零；滤波只出现在 rps 路径；`get_state` 返回同拍完整快照
- [ ] `output_invert` 与 `encoder_invert` 独立；低占空比方向自检确认命令、rps、position 符号一致
- [ ] 稳定槽位映射表与 `PLATFORM_PMOTOR_ID_MAX` 静态断言绑定；disabled 槽位返回 `PLATFORM_ERR_NOT_SUPPORTED`
- [ ] driver 私有 PID/滤波/死区在 PC 上单测通过，各电机实例状态互不共享
- [ ] 跨计数器溢出边界，position 连续无跳变

- [ ] 底层错误锁存 fault/state，并在同一故障路径 best-effort 失能全部电机；具体向上回调可后续扩展
- [ ] 控制回调 ISR 无日志、无阻塞，最大启用电机数下实测 WCET/抖动并留有裕量
